<p>&lt;p&gt;&amp;nbsp;&lt;/p&gt;</p>
<p>&lt;p&gt;To generate a Kerberos ticket on each Informatica node, do the following:&lt;/p&gt;</p>
<p>&lt;ol&gt;</p>
<p>&lt;li&gt;Ensure that the Kerberos utilities are already installed on the Informatica node(s).&lt;/li&gt;</p>
<p>&lt;li&gt;Use the Kerberos&amp;nbsp;&lt;em&gt;kinit&lt;/em&gt;&amp;nbsp;utility to generate the credential cache file (token) for the user account that runs Informatica Services.&lt;/li&gt;</p>
<p>&lt;li&gt;Run the&amp;nbsp;&lt;em&gt;kinit&lt;/em&gt;&amp;nbsp;utility, and specify the User Principal name that should be used to authenticate. &lt;br /&gt;Typically, this name should match the user running Informatica services.&lt;/li&gt;</p>
<p>&lt;li&gt;Use the&amp;nbsp;&lt;em&gt;forwardable (&lt;strong&gt;-f&lt;/strong&gt;)&lt;/em&gt; option&amp;nbsp;while creating the user credential cache. &amp;nbsp;&lt;/li&gt;</p>
<p>&lt;li&gt;Syntax of the &lt;em&gt;kinit&lt;/em&gt; command is:&amp;nbsp;&lt;em&gt;&lt;br /&gt;kinit -f &amp;lt;principal name&amp;gt;&lt;/em&gt;&lt;/li&gt;</p>
<p>&lt;li&gt;The format for the principal name is:&amp;nbsp;&lt;em&gt;&lt;br /&gt;&amp;lt;username&amp;gt;@&amp;lt;realmname.com&amp;gt;&lt;/em&gt;. &lt;br /&gt;Enter the realm name in uppercase letters.&amp;nbsp;&lt;/li&gt;</p>
<p>&lt;li&gt;If &lt;em&gt;&amp;lt;realmname.com&amp;gt;&lt;/em&gt; is not specified, select the default realm from the &lt;em&gt;krb5.conf (.ini)&lt;/em&gt;.&lt;/li&gt;</p>
<p>&lt;li&gt;Enter the password for the user account.&amp;nbsp;&lt;/li&gt;</p>
<p>&lt;li&gt;If the &lt;em&gt;KRB5CCNAME&lt;/em&gt; environment variable is set before running the &lt;em&gt;kinit&lt;/em&gt; utility, &lt;em&gt;kinit&lt;/em&gt; writes to the credential cache in the location specified in the environment variable, otherwise it&amp;nbsp;writes to the default credential cache file or the file specified with &lt;em&gt;-c&lt;/em&gt; option.&lt;/li&gt;</p>
<p>&lt;li&gt;Run the&amp;nbsp;&lt;em&gt;klist&amp;nbsp;&lt;/em&gt;command to verify that you have a valid token available.&lt;/li&gt;</p>
<p>&lt;/ol&gt;</p>
<p>&lt;p&gt;&amp;nbsp;&lt;/p&gt;</p>
<p>&lt;p&gt;For the same node, edit the PowerCenter configuration file:&lt;/p&gt;</p>
<p>&lt;ol&gt;</p>
<p>&lt;li&gt;Go to &lt;em&gt;$INFA_HOME/services/shared/hadoop/&amp;lt;your hadoop distribution name&amp;gt;/conf&lt;/em&gt;&lt;/li&gt;</p>
<p>&lt;li&gt;Edit the&amp;nbsp;&lt;em&gt;core-site.xml&lt;/em&gt;&amp;nbsp;file and search for the &lt;em&gt;hadoop.security.kerberos.ticket.cache.path&lt;/em&gt;&amp;nbsp;property.&lt;/li&gt;</p>
<p>&lt;li&gt;Update the value with the Ticket Cache file name above (for instance: &lt;em&gt;/tmp/krb5cc_121299384_NQ2HB7&lt;/em&gt;).&lt;/li&gt;</p>
<p>&lt;li&gt;Save the file.&lt;/li&gt;</p>
<p>&lt;li&gt;Repeat the steps above on all the Informatica nodes.&lt;br /&gt;&lt;em&gt;&lt;br /&gt;Note&lt;br /&gt;&lt;/em&gt;Refer the &lt;em&gt;Big Data Edition Installation&lt;/em&gt; and &lt;em&gt;Configuration Guide&lt;/em&gt; for any specific steps related to your Hadoop distribution.&lt;/li&gt;</p>
<p>&lt;li&gt;Restart your PowerCenter Integration Service for the changes to take effect.&lt;br /&gt;&lt;em&gt;&lt;br /&gt;Note&lt;br /&gt;&lt;/em&gt;To avoid disintegration of PowerCenter with cluster, ensure an automated script is scheduled to renew the token before it expires. Contact your Hadoop administrator for more information.&lt;/li&gt;</p>
