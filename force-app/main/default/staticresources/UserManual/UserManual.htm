Basic configuration

 

This module allows the automatic creation of Managed Clauses in any object of the system that is configured for it. Creating/deleting Managed Clauses will occur when you insert, update, or delete a record.

Here are the steps to follow if you would like to configure the automatic management of Managed Clauses for the standard object Contract:

1)   Create in the Managed Clause object (APXT_Redlining__Managed_Clause__c) a Lookup field to Contract:

 

 

2)   Modify the Contract Page Layout to add additional fields to the Managed Clauses Related List that allow for easy identification of added clauses:

3)   Configure the corresponding Clause Bundle (if they are not already configured). If there is more than one Clause Bundle that can be associated with the Contract object, it is important to set the order in which you want the Managed Clause records associated with Contract to be generated. To do this, the installation package adds the Order field to the Clause Bundle object in the Conga Contracts package:

 

 

NOTE: If this field is not in the Page Layout, add it to both the Page Layout and the List View to make it easier to set up.

4)   Add Clause records to the Clause Bundle (if not already added). Clause's records have their own order within the Clause Bundle. This order will be respected but will be subject to the Clause Bundle order.

 

5)   Configure the Clause Bundle Mapping. This consists of creating a record in the Custom Metadata Type Clause Bundle Mapping where the object to which the Managed Clause records will be associated will be set. It will also allow you to set values for other configuration parameters. The steps to follow are:

 

a.    Go to Setup -> Develop -> Custom Metadata Types -> Clause Bundle Mapping -> Manage Clause Bundle Mapping -> New. A form will appear to configure the Clause Bundle Mapping:

 

 

b.    All required fields must be completed here. The meaning of each field is described below:

 

                                         i.    Label = Identifier for the field. It is recommended to place the name of the object and any additional identifier.

 

                                        ii.    Clause Bundle Mapping Name = Leave the default name assigned automatically when you change the Label value.

                                       iii.    Object = Here you must select the object to which the Managed Clause records will be added for each of the existing Clause records in the selected Clause Bundle. As we are doing a configuration for Contract, we select this object.

                                      iv.    Object Control Field = Field of the selected object that will control the execution or not of the Clause Bundle selection process and Managed Clause association. It is optional and must be a Boolean type field. Its value will determine the execution of the process. If no value is set for this field, the process is always executed. For our configuration, leave the value of the field at "--None--".

                                        v.    Managed Clause Object = Here we must always select the object Managed Clause [APXT-Redlining].

                                      vi.    Managed Clause Lookup Field = Here we must select the Lookup type field of the Managed Clause object pointing to Contract (previously created). For our configuration, select the field Contract.

                                     vii.    Operator = Logical operator to be used to evaluate the Clause Bundle selection criteria if more than one selection criterion exists. The selection criteria will be configured later. For our configuration, we will leave the default value "AND".

                                    viii.    Protected Component = Leave this field with false value.

                                      ix.    Allow Multiple Clause Bundles = Setting this parameter to true will allow Managed Clauses records associated with Clauses of more than one Clause Bundle to be associated with Contract, provided that the selection criteria allow this. In our configuration, we will leave the value of this field in false, which will make that only Managed Clause records originating from a single Clause Bundle are associated to Contract.

                                        x.    Active = Enables you to activate/deactivate the mapping execution. For our configuration, we will leave the value of this field in true.

                                      xi.    Test Case = It should always be left on false.

Save the record using the Save button.

6)   Configure Clause Bundle Mapping Parameters. These will be the criteria for the selection of the Clause Bundle. To do this, we must follow these steps:

a.    Go to Setup -> Develop -> Custom Metadata Types -> Clause Bundle Mapping Parameter -> Manage Clause Bundle Mapping Parameters -> New. A form appears for setting the criterion with the following fields:

                                         i.    Label = It is recommended to place something that describes the selection field. In our configuration, we will place "Contract - Description".

                                        ii.    Clause Bundle Mapping Parameter Name = Leave the value that is set automatically when the Label field is filled.

                                       iii.    Clause Bundle Mapping = Here we must select the Clause Bundle Mapping record created in the previous step, i.e. "Contract_Bundle_1".

                                      iv.    Type = Determines the type of parameter. The options are:

 

1.   Bundle Selection Criteria = Allows you to specify a selection condition for the Clause Bundle.

 

2.   Source Object Assignment = Allows you to specify the updating of the value of a Contract field with the value of a selected Clause Bundle field. If more than one Clause Bundle is selected, the value that will prevail will be that of the last Clause Bundle processed according to your order.

 

For our configuration, we will use the default option "Bundle Selection Criteria".

                                        v.    Source Object = Object for mapping. In our case, the source object is Contract.

                                      vi.    Source Field = Field of the mapping object to be compared with the field of the Clause Bundle to check if they are the same (meets selection criteria). In our configuration, we will select the Description field.

                                     vii.    Target Object = When the parameter type is "Bundle Selection Criteria", the object Clause Bundle must always be selected. This applies to our configuration, so we select here the object Clause Bundle [APXT_Redlining].

                                    viii.    Target Field = Clause Bundle field whose value will be compared with the field of the source object and will determine whether the Clause Bundle is selected. For our configuration, we will select here the Description field.

                                      ix.    Value = It is used in cases where we want to search for a specific value in a field of the Clause Bundle. In our case, we will compare the values of the source/destination fields, so we will not use this parameter. Leave it blank.

                                        x.    Required = Determines whether the field value of the target object should be different from null (or true for checkboxes). We activate it so that Clause Bundle records whose Description field equals null are not considered.

                                      xi.    Protected Component = Always leave on false.

                                     xii.    Active = Set to true for the criterion to be evaluated.

7)   Create a trigger on the Contract object. To do this, follow these steps:

a.    Go to User Name -> Developer Console -> File -> New -> Apex Trigger.

b.    Specify the name and object for the trigger. In our case:

                                         i.    Name = MCC_Contract

                                        ii.    sObject = Contract

c.    Replace the trigger text with the following:

trigger MCC_Contract on Contract (before delete, after insert, after update){

ManagedClauseCreationTriggerHandler handler = new ManagedClauseCreationTriggerHandler(Schema.SObjectType.Contract.getName());

handler.onTrigger(Trigger.newMap, Trigger.oldMap);

}

 

d.    Click File -> Save.

8)   Test the configuration and verify that it is working as expected. To do this, the following actions must be carried out:

a.    Go to Sales -> Contracts -> New.

b.    Fill in the required fields. For the moment, leave the Description field empty. The form should look like this:



c.    Save the record and verify that no Managed Clause records have been associated with the corresponding Related List.

d.    Edit the Description field value, assigning it the text "DEMO" (must match the description of the Clause Bundle to be selected) and save the record.

e.    As many Managed Clauses as there are Clauses with the Clause Bundle configured must have been associated to the Contract register. In our case, we only had two Clauses in the Clause Bundle:

 



f.     If the Contract record is modified again and the value in the Description field is removed, the associated Managed Clauses must also be removed.

g.    If the value "DEMO" is set in the Description field again, the corresponding Managed Clauses must be associated with the Contract record.

 

Considerations

 

With the above configuration the correct operation of the package can be confirmed after installation. The configuration options will depend on each case of use. In general, you will need to add new fields to the Clause Bundle object to set the selection criteria.

It is also possible to configure the updating of the value of a field of the object to which the Managed Clause records are associated using the value of the selected Clause Bundle field. To do this, when configuring the Clause Bundle Mapping Parameters, the record type must be changed to "Source Object Assignment". In this case, the Source Object must be the Clause Bundle and the Target Object the object for the mapping (Contract in our previous example).

Finally, it is possible to change the persistence behavior of Managed Clause records, as described in the following point.

 

Default behavior change

 

With the previous configuration we were able to confirm the basic operation of the module. Whenever the value of a field that is part of a selection criterion is changed in a Contract record, the selection criteria set in the Clause Bundles will be evaluated to see if it is necessary to modify the Managed Clauses. By default, if there is already a Managed Clause associated with Contract and a change in the selection criteria would cause that Managed Clause to be added again, the existing one will remain unchanged. In case you want to change this behavior, so that whenever you change the values of the fields used as selection criteria for the Clause Bundles are deleted and overwrite the existing Managed Clauses, you can change the value of the Always Delete Existing Managed Clauses field to true. To do this, follow these steps

1)   Go to Setup -> Develop -> Custom Settings -> Managed Clauses Settings -> Manage.

2)   Click New to add a default value for the organization (if none exists) and activate the Always Delete Existing Managed Clauses check mark:



3)   Save the record.

In this way, whenever there is a variation in the fields that are part of the Clause Bundles selection criteria, all existing Managed Clauses will be removed before the new Managed Clauses of the selected Clause Bundles are added.
